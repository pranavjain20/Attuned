# Day 12 — Continuous Intelligence + Komal's First Playlist

Biggest architecture change since the matching engine rewrite. Replaced the state machine (7 discrete states → static profile → modifiers) with a continuous weighted function. 12 physiological z-scores (recovery, HRV, RHR, sleep stages, debt, trends, deltas) feed directly into the neuro profile. State classifier demoted to display labels only.

Key discoveries:
- Initial weights were too aggressive (Para 0.64 for a 44% day — accumulated fatigue territory for "baseline but tired"). Root cause: weights calibrated per-signal but 12 correlated signals stack multiplicatively. Fixed with WEIGHT_SENSITIVITY = 0.20.
- Neuro scoring had 10 songs tied at 1.000 — one-sided dot product normalization couldn't discriminate when profile had moderate values. Switched to proper cosine similarity.
- Restorative sleep gate was using mean - 1*SD threshold ("not deficit") instead of mean ("above average"). Deep sleep at 16th percentile was counting as "restorative."

Playlist rotation overhauled: days-since-last-appearance replaces consecutive streak counter. Freshness uses latest playlist per day only (draft iterations no longer pollute). Bollywood motivational filter excludes scene-tied songs from non-peak playlists.

Komal's first live playlist shipped — "Rest & Repair" for 47% recovery. The old state machine gave full calming mode (Kun Faya Kun, The Night We Met). The new continuous profile saw her HRV was fine and debt was low, producing balanced romantic music (Tum Se Hi, Shayad, Earned It). Same recovery score, completely different playlist.

1,048 tests passing.
